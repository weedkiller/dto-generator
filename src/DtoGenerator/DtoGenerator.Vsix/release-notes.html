Release notes

Version 1.8

Bugs fixed:

  • Fixed issue with overriden properties.
  • Fixed issue when choosing a destination project in a solution with several similar project names
  • Fixed issue when having another selector expression within custom code in DTO, and DTO is regenerated

Version 1.7

Features added:

  • Added option to preserve ComponentModel.DataAnnotations attributes (thanks to Gwenou).
  • When new DTO is generated, it is opened automatically.

Bugs fixed:

  • Fixed bug System.Guid and similar simple types.
  • Fixed bug with nested entities (for now they are just ignored)

Version 1.5

Features added:

  • Enabled generating DataContract and DataMember attributes if chosen.

Bugs fixed:

  • Fixed bug with using .NET core and wrongly generated namespaces.
  • Fixed bug with unknown namespace if mapper is not selected.

Features added:

  • Detailed stack trace report opens in browser.

Version 1.4

Bugs fixed:

  • Fixed bug in generating related entity properties in selector expression. Replaced null with default(type).
  • Fixed bug in regenerating DTO to remember if DTO was inherited or not.

Features added:

  • Added option to generate only DTO (not generate mapper at all)

Version 1.3

Bugs fixed:

  • Fixed bug when using Nullable<System.DateTime> and similar types with explicit Nullable generic type name.
  • Fixed bug when generating initial DTO on completely new project.

Version 1.1

Bugs fixed:

  • Fixed bug when regenerating DTO to auto selected already generated properties.

Changes:

  • Now DTO can be named in any way, with some recommended naming.
  • It is possible to choose if base type will be introduced
  • It is easier to pick DTO location
  • Message box is presented if DTO is about to be regenerated
